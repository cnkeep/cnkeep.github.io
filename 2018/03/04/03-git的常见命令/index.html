Git的简单使用

前言

  开发过程当中难以避免的要使用到版本控制工具，以前使用的SVN, 但随着Git的出现，它以及其优良的特性迅速火热，本节我们就来简单了解一下。

介绍

  Git是一个开源的分布式版本控制系统(本质是一个内容寻址文件系统，后面章节会做介绍)，用于敏捷高效地处理任何或小或大的项目。
Git 与常用的版本控制工具 CVS, Subversion 等不同，它采用了分布式版本库的方式，不必服务器端软件支持（这点很重要，他可以在服务端无法访问的
时候提交到本地仓库，等网络恢复后推送到远程仓库）。

Git 与 SVN的区别

简单使用

初始化命令

init 初始化仓库

clone 克隆仓库

config 配置

remote 远程仓库

###　基础命令

status 查看文件状态

add 暂存已修改文件

commit 提交更新

push 推送到远程仓库

fetch 从远程仓库拉取数据

merge 合并分支

pull ( fetch + merge )

常用命令

diff 查看修改

log 查看提交历史

撤销操作

checkout

tag 打标签

rm 移除文件

分支命令

branch 创建分支

checkout 切换分支

底层命令

cat-file 读取 GIT 仓库对象

gc 生成包文件

查看对象 

$ find .git/object/ -type f     #所有对象列表
$ git rev-list --objects --all  #blob列表
